- (direction) Inputs structured + pre-sorted by upstream BMad skills (PRD, DESIGN.md, EXPERIENCE.md, ARCHITECTURE-SPINE.md). Extract directly, no elicitation. PRD+addendum absorbed as sources; DESIGN/EXPERIENCE/ARCH adopted as companions (the HOW).
- (capability) CAP-1 Open & render a paper: load PDF from disk, render pages, navigate via page controls + table of contents. Success: a multi-page PDF opens and every page renders with working ToC jump.
- (capability) CAP-2 Read controls: smooth vertical scroll, zoom (ctrl+/-, ctrl+scroll, buttons, live %), hand/space pan. Success: scroll/zoom/pan stay fluid (~60fps) on a 50+ page paper.
- (capability) CAP-3 Annotate: highlight, underline, pen/brush freehand, free-floating textbox memo, comment (highlights text + anchored bubble pin), box/region select. Success: each annotation type can be created and appears anchored to the right spot on the page.
- (capability) CAP-4 Drag-to-annotate + contextual quick-box: drag-select creates a mark; on release a mode-specific quick-box pops (selection->tool picker; highlight/underline->color; pen->color+width; memo->text; comment->bubble). Success: user creates and tunes an annotation without returning to the left rail.
- (capability) CAP-5 Edit annotations: select, move/resize/restyle/re-edit text, undo/redo, delete. Success: an existing annotation can be moved/recolored/deleted and undo/redo reverses each edit.
- (capability) CAP-6 Annotation Bank: toggleable panel listing all annotations; click jumps to the annotation and flashes it. Success: every annotation appears in the list and clicking a row scrolls the canvas to it.
- (capability) CAP-7 Persistence: autosave annotations to disk; restore exactly on reopen. Success: after closing and reopening a paper, every annotation is present at its correct position across zoom levels; nothing silently lost.
- (constraint) PDF canvas is pixel-stable: rail, Annotation Bank, and quick-boxes all OVERLAY, never reflow/resize the page (NFR-1). The defining quality bar.
- (constraint) All annotation geometry stored page-normalized (fractions [0,1], top-left, against rendered page box); screen position always derived, never persisted (NFR-3, ARCH AD-4).
- (constraint) Annotations persist on disk under ~/.paper-mate/library/{doc_id}/ (doc_id=SHA-256 of PDF bytes), separate from source.pdf which is never modified (ARCH AD-8, NFR-4).
- (constraint) Runs in Chrome AND Firefox as a localhost SPA + dockerized FastAPI backend that owns all disk I/O; client never touches the filesystem (ARCH AD-1). Forced by Firefox lacking local File System Access.
- (constraint) Read experience is immersive/non-distracting: minimal Obsidian-style chrome, DESIGN.md tokens, UI recedes behind the paper (NFR-5).
- (constraint) AI (Phase 3, directional) uses LOCAL CLI agents only (Claude/Codex/Antigravity), no hosted API, behind one switchable interface.
- (direction) Non-goal: no collaboration/sharing.
- (direction) Non-goal: no cloud sync / WebDAV / import-export in v1.
- (direction) Non-goal: no mobile/tablet (desktop web only).
- (direction) Non-goal: no reference-manager (Zotero-style).
- (direction) Non-goal: Phase 2 (reading helper: inline ref previews, metadata, export, Library) and Phase 3 (AI companion) are out of the v1 BUILD; directional only, architecture reserves seams.
