- (capability) CAP-1 Collection & table (FR-1..6). intent: on boot the user lands in the Library, a table of every paper (Title, Authors, Added, File type, total count) they can multi-select and whose columns/sort/filter they control. success: booting opens the table with the four columns + 'N files' count; user toggles a column, sorts any column asc/desc, filters by column value, multi-selects via checkboxes.
- (capability) CAP-2 Add papers (FR-7..11). intent: user adds one or more PDFs in a single bulk action; auto-extract Title+Authors (embedded metadata + text), optional external enrich, inline-correctable; a paper is never lost to a failed parse. success: dropping N PDFs adds N rows without freezing; most rows show correct Title/Authors; a PDF yielding nothing still enters (filename title, editable); offline/failing enrich keeps local fields + a non-error 'enrichment skipped' notice.
- (capability) CAP-3 Folders (FR-12..16). intent: organize papers into nested custom folders (create/rename/delete), each paper in <=1 folder (unfiled show under All/Uncategorized), move papers or multi-selections between folders, filter the table to a selected folder. success: nested folders CRUD; moving a paper (and a multi-selection) into one works; selecting a folder filters the table; deleting a folder re-homes its papers to Uncategorized without deleting them.
- (capability) CAP-4 Open in annotator (FR-18..20). intent: user opens a paper into the annotator by double-clicking its row (not an ad-hoc disk picker); its stable identity restores existing annotations; user can return to the Library from the reader. success: double-click a row enters the annotator on that paper with prior annotations intact; a control returns to the Library.
- (capability) CAP-5 Persistence (FR-21). intent: the collection, folder structure, per-paper metadata, and folder assignments persist under ~/.paper-mate and survive restart. success: after restarting, the same papers, folders, metadata, and assignments are present.
- (capability) CAP-6 Trash / delete (FR-22..24). intent: user soft-deletes a paper (or multi-selection) to a Trash view that retains its annotations, restores a trashed paper, or purges it to permanently remove the paper and its annotations. success: delete moves to Trash with annotations retained; restore returns it; purge permanently removes the paper and its annotations.
- (capability) CAP-7 Note file-type reserved (FR-17). intent: the data model and table support a 'Note' file-type distinct from PDF documents, displayed but not authored in-app this sprint. success: the table can display a Note-typed entry distinct from a PDF; no in-app note authoring exists.
- (constraint) Local-first (NFR-1): every Library feature works fully offline; the external metadata lookup is the only network call, opt-in, degrades gracefully (never blocks the add).
- (constraint) Non-blocking add (NFR-3): a bulk upload never freezes the table; extraction runs off the interaction path, rows appear as they resolve, browsing continues during extraction. See spine AD-L2/AD-L4.
- (constraint) Collection scale (NFR-4): for hundreds of papers, sort/filter/scroll act without a visible stall. See AD-L1 display cache.
- (constraint) Durable, forward-compatible store (NFR-5): Library metadata + folder structure persist under ~/.paper-mate in an additive-tolerant format; a breaking schema change is an AD-8-class persisted-format break -> MAJOR version bump.
- (constraint) Safe copy-in (NFR-6): the uploaded PDF is copied into the collection; the copy never corrupts/loses the original, and a mid-copy failure leaves the collection consistent. See AD-L4.
- (constraint) Same store + identity: a Library paper reuses the existing ~/.paper-mate/library/{doc_id}/ store and the doc-scoped annotation store; doc_id = SHA-256 of PDF bytes is both its identity and its annotation key. Inherited AD-8; see spine.
- (constraint) No auth, localhost single-user (NFR-2 / inherited AD-10): no accounts, no multi-user, no CORS.
- (non-goal) Remote sync / WebDAV / Google Drive this sprint (F8): a planned follow-on epic; the architecture reserves the switchable-backend seam (mirroring the agent abstraction) but it is NOT built here.
- (non-goal) In-app note authoring: Note is a reserved, displayed file-type only (CAP-7); authoring is out of scope this sprint.
- (non-goal) Global Search nav, Chats tab, full-text indexing ('Full text: Available'), viewed/last-opened tracking (PRD Out of Scope).
- (non-goal) No change to the annotation model or the reader/annotator itself: the Library only routes into the existing annotator (Epics 1-5); no new annotation entity.
