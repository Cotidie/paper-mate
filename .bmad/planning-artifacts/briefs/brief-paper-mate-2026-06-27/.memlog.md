- (decision) Stakes: personal paper-reading + e-textbook learning for self and fellow grad students. Passion/learning project -> right-size brief, low rigor.
- (decision) Scope: v1 = Phase 1 (PDF viewer/annotator) ONLY. Phase 2 (reading helper) and Phase 3 (AI companion) are vision/roadmap, out of v1 scope.
- (decision) Differentiator confirmed: local-agent AI (own keys, no cloud/per-token cost) + truly immersive Obsidian-minimal reading. AI lands in Phase 3; immersive reading is the v1 hook.
- (decision) Working mode: Fast path. Draft full brief with [ASSUMPTION] tags, user corrects in review.
- (override) Working mode CHANGED Fast -> Coaching path (user interrupted to correct). Walk section by section, pull from user, push where thin. No batch-draft.
- (decision) Problem section approved. Core v1 pain = annotation actions reflow the PDF (highlight/comment/side-panel all resize page), modal tool-switching, no table-of-contents nav. Incumbent = Kami (Chrome). AI moved to Vision, not v1 problem.
- (decision) Core mechanic: annotations render as overlay layer on the PDF; comment composer is a popover anchored to the spot; Annotation Bank overlays or toggles fully off. Invariant: PDF area never changes size from any annotation action.
- (decision) Annotation interaction model = Hybrid: non-modal floating selection menu (select text -> highlight/underline/comment) AND left toolbox drawer for persistent/power tools. Safe default avoids stray-drag comment boxes.
- (decision) Persistence: annotations stored locally (browser/local files), private by default, no account, no cloud. Privacy is a positioning angle alongside local-AI.
- (decision) Keyboard-first: each tool/mode gets a single-key shortcut (proposed V select, H highlight, U underline, C comment, T toc, B toggle bank, [ toggle drawer, Ctrl+/- zoom, Esc to select). Reinforces immersive keyboard-driven reading. Exact keys taggable in PRD.
- (decision) Persistence/reopen model: identify PDF by content-hash fingerprint (rename-safe); anchor each annotation to page + normalized position + quoted text (zoom-safe re-resolve). Storage = Option B: sidecar JSON file next to the PDF via File System Access API (chosen over IndexedDB because clearing browser data would wipe IndexedDB). Chromium-only, portable, backup-able. Annotated-PDF export remains the share format.
- (decision) Scope confirmed. IN v1: open single local PDF (pick each time, no library); overlay highlight/underline/text-comment; drag-to-annotate + hybrid select-menu + toolbox drawer; per-tool keyboard shortcuts; toggleable drawer + Annotation Bank; table of contents; zoom; sidecar persistence w/ reopen restore; export annotated PDF. OUT (later phases): library/folders, image/ink comments, notes/markdown export, accounts, sync, collaboration, all AI, figure/footnote/reference previews, metadata extraction.
- (decision) Success Criteria approved (personal-project framing): dogfood test (builder drops Kami, daily driver), immersion holds (read+annotate full paper, page never resizes), zero annotation loss on reopen across restarts, peer pull (handful of grad students adopt unprompted). No vanity metrics.
