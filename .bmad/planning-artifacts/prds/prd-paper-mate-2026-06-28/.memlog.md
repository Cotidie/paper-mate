- (decision) Stakes: personal/solo tool, not internal or commercial. PRD stays lean (~2pg target).
- (decision) Viewer/annotator reference = Kami. Gaps to fix vs Kami: annotations must NOT reflow/change PDF area layout; drag-to-change-tool support; textbox memo + pen/brush freehand drawing.
- (decision) Core long-term vision = integration with LOCAL AI agents (next version). Phase 1 viewer/annotator is foundation; AI companion is the real payoff.
- (decision) Working mode: Coaching path, Vision+Features entry. Section order = Vision/Goals -> Features(FRs) -> NFRs -> scope/phasing.
- (decision) Success signal (solo): 'stopped opening Kami' = annotations persist/reserved, easy to edit annotations, smooth scroll+zoom, ZERO layout change of PDF area.
- (decision) Kami features to KEEP: undo (history), range/area-selection tool (box select region), hand tool (pan PDF by drag). Plus highlight/underline/comment/ToC already in scope.
- (assumption) Non-goals NOT yet decided (collab/sharing, cloud sync, mobile, ref-manager). OPEN ITEM to revisit at finalize.
- (decision) drag-to-change-tool CONFIRMED: drag-select region/text -> quick tool picker pops (highlight/underline/comment/memo) without returning to left rail.
- (decision) Comment vs memo CONFIRMED: comment=pinned note anchored to a spot (opens on click); textbox memo=free-floating text typed onto page.
- (decision) Annotation Bank v1 = separate layout listing all annotations + click-to-jump. No filter/edit-in-bank/export/search in v1.
- (decision) Feature groups v1: FG-A Viewer(layout-stable), FG-B Tools(highlight/underline/pen-brush/textbox-memo/comment/box-select), FG-C Interaction(drag-to-annotate+drag-to-change-tool, edit, undo/redo, delete), FG-D Annotation Bank, FG-E Persistence.
- (decision) Runtime: Chrome + Firefox desktop. Mobile/tablet out of v1.
- (decision) Storage: local-first ON DISK. Dedicated folder e.g. ~/.paper-mate/ holding uploaded PDFs + their annotations side by side. IndexedDB = nice-to-have. Sync/WebDAV/cloud/import-export = LATER.
- (override) NFR list v1: NFR-1 layout stability(defining bar), NFR-2 smoothness ~60fps on 50+pg, NFR-3 anchor fidelity across zoom, NFR-4 durability/local-first, NFR-5 immersion(min chrome).
- (assumption) TENSION/OPEN: writing to ~/.paper-mate on disk from pure web needs File System Access API (Chrome ok, Firefox NOT supported) OR a local backend/desktop shell. Conflicts with Firefox-support requirement. Resolve in architecture; NOTE FOR PM.
- (decision) v1 = Phase 1 only. Phase 2/3 captured as lightweight roadmap in this PRD (so v1 architecture reserves room for spatial-anchor + agent abstraction).
- (decision) vNext north star confirmed (as-is): Phase 3 = local CLI agents only (no hosted API), vendor-switchable (Claude/Codex/Antigravity), paper digest auto-injected into context.
- (event) PRD finalized
