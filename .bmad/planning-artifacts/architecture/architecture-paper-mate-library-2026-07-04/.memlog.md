- (decision) AD-L4 Bulk-add flow & idempotent upload [chosen by Wonseok]. Binds FR-7,8,10; NFR-3,6; AD-8. Prevents: table freeze on bulk add; ambiguous partial-failure; duplicate rows for one paper; a paper lost to failed parse. Rule: (1) Upload = one POST /api/library per PDF, client-throttled (concurrency cap ~4); each returns an optimistic row immediately (docId, title=filename, status:extracting); rows stream into the table as requests land (NFR-3). (2) Every paper carries an extraction status: extracting -> ready | enrich-skipped | parse-failed; client POLLS the collection endpoint until all statuses settle, then stops. (3) Failure splits: store-failure (not a PDF / disk error) rejects that one file with a per-file notice, others unaffected; parse-failure still enters the paper as filename-title, status:parse-failed, editable (FR-10) — never lost. (4) Idempotent dedupe by docId (AD-8): a re-upload resolving to an existing {doc_id}/ creates NO duplicate row and returns the existing paper; if trashed, the re-upload RESTORES it from Trash (honor add-intent), notice 'restored from Trash'; existing annotations.json/meta.json never overwritten. (5) Safe copy-in (NFR-6): storage writes source.pdf atomically (temp+rename) so a mid-copy failure leaves the collection consistent and never corrupts the original.
- (decision) AD-L5 Trash & folder lifecycle [chosen by Wonseok]. Binds FR-12,13,16,22,23,24; RATIFIES PRD assumption A1. Prevents: annotation loss on delete; ambiguous restore target; orphaned folder subtrees; papers deleted with their folder. Rule: (1) Soft-delete (FR-22): flip trashed in library.json; annotations untouched; paper leaves normal/folder views, shows only in Trash filter; paper RETAINS folder membership while trashed. (2) Restore (FR-23): clear trashed; paper returns to its remembered folder; if that folder no longer exists, lands in Uncategorized. (3) Purge (FR-24): permanent — delete the whole {doc_id}/ dir (source.pdf+annotations.json+meta.json) + its library.json entry; annotations gone with it; manual only, no auto-purge. (4) Delete folder (FR-16, ratifies A1): deletes that folder AND all descendant folders (whole subtree); every paper anywhere in the subtree -> Uncategorized; papers are NEVER deleted by a folder delete. (5) Each paper belongs to <=1 folder (FR-13); membership authoritative in library.json (AD-L1).
