## What went well

- **Live smoke caught three of the epic's most dangerous bugs — all invisible to the full green unit suites.** (a) 6.1: navigating back-to-Library unmounted a fully-loaded `<Reader>` and crashed the ENTIRE app into React Router's error boundary (`TypeError: pdf.destroy is not a function`) — dead code exposed for the first time because routing made unmount a real, reachable path; `PDFDocumentProxy` has no `.destroy()` in pdfjs-dist 6.0.227, fixed to `pdf.loadingTask.destroy()`. (b) 6.4: React StrictMode's dev-only double-invoke latched both `mountedRef` guards to `false`, so every settled upload was silently dropped — yet all 897 unit tests passed, because jsdom's `render()` is not StrictMode-wrapped. (c) 6.5: Crossref `rows=1` always returns a top result, so a bare `items[0]` "corrected" COCO to an unrelated keyword-spam paper; fixed with a token-set Jaccard ≥ 0.5 title-plausibility gate (the DOI path stays exact). Treat "all unit tests green" as necessary, never sufficient.

- **Cross-model Codex review (AE-6) kept catching real bugs, one HIGH and several Medium.** 6.1 HIGH: `useAutosave`'s unmount cleanup only *cancelled* the pending debounce, so an edit made just before back-to-Library was silently dropped — fixed to *flush* on unmount (the flush reads target/snapshot live from the store and completes independent of mount state). 6.2 Medium: a malformed `library.json` row escaped the `StorageError` taxonomy as a raw `KeyError` and could crash boot `lifespan`; consolidated row validation at the single `_read_index_unlocked` choke point. 6.2 Low: the `lifespan → reconcile_library` wiring was untested (a module-level `TestClient` never runs lifespan) — added a `with TestClient(app)` boot test. 6.4 Medium: a stale library fetch could clobber newer optimistic state — monotonic `fetchSeqRef` guard. 6.5: 3 Medium + 1 Low, all resolved. 6.8: 2 Medium — the `enrich` facade could leak an *injected* enricher's exception past the port's "never raises" contract, and the AST purity guard false-greened on `from app import storage`; both fixed with regression tests.

- **A deferred debt item closed ON PURPOSE, ahead of the feature that needed it.** AE-4/AE3-3 (doc-scope the store) was punted through all of Epic 3, then flagged. It was actually closed by Story 5.8 (PR #41) BEFORE Epic 6 — and Story 6.7's doc-switch isolation (open A, annotate, back, open B → B shows none of A's marks; re-open A → exactly A's) is the direct payoff, verified live at DPR=2 with a real cross-page highlight. This is the exact opposite of Epic 3's "defer with a plausible reason, get bitten anyway."

- **The "foundation pays off, tools become thin" pattern repeated.** 6.3's table is a thin render over 6.2's one-read index; 6.7's open-with-annotations is a thin consumer of the 5.8 doc-scoped store — zero new persistence machinery. Same #1 win Epics 2 and 3 named.

- **Structural-refactor-as-its-own-story (6.8) closed the epic on a clean footing** — echoing Story 5.0 before Epic 3. The 621-line `storage/__init__.py` god-module split into a focused package behind a byte-identical facade; extract/enrich separated behind a Crossref `Enricher` port; the `routes/docs.py` error-envelope + exception mapping deduped into `routes/_errors.py`; the client `library/` moved onto the scaffold-react `<Name>/` layout. Contract byte-identical (`schema.d.ts` unchanged), all suites green. The next epic builds on order, not sprawl.

- **Bookkeeping discipline recovered from Epic 3's low.** Every one of the 8 stories has a COMPLETE Dev Agent Record (model, debug log, completion notes, review outcome) — the direct fix of Epic 3's named weak link (blank 3.2 record). AE3-1 (sprint-status at merge) held, including this epic's own 6.8 flip. AE3-6 (version/lockfile match) held, with `test_version.py` in the suite and 6.8's `uv.lock` re-run.

- **The "mock everything the mount calls" rule was named and applied consistently** as a clean extension of CLAUDE.md's "keep the render/ mocks in sync" principle. Each fetch-on-mount story (6.3, 6.4, 6.5, 6.6) carried an explicit "REGRESSION FIRST" heads-up and updated every mounting test. ## Recurring problems / patterns

1. **The AE-5 DPR>1 smoke carry-over survived a SECOND retro unclosed — and Epic 6 revealed WHY.** Stories 6.3/6.4/6.5 each correctly declared themselves "not a geometry feature, AE-5 does not apply" (they render no PDF coordinates). But 6.7, the one real geometry story, hit the carried-over problem head-on: coordinate-drag cross-page selection repeatedly mis-hit (an arXiv sidebar-watermark span; then a 2-column paper whose pdf.js DOM spans emit in column order, so "bottom-most by Y" is not "last in DOM"), and even raw `mouse.move/down/up` was non-deterministic across a scrolled multi-page boundary (native auto-scroll-while-selecting raced the app's live-window logic). The product mechanism (`collectTextRects` per-page split) was verified correct every time — no leak — but final verification had to fall back to a programmatic `Range` + trusted `pointerup`. So AE3-5 (backfill 2.11 box-select + 3.3 cross-page grouped-delete DPR smoke) is STILL unconfirmed, and we now know cross-page real-drag is genuinely un-automatable in this harness, not merely un-prioritized. Extends `[[drag-tools-dont-create-text-selection]]`.

2. **The claude-in-chrome extension was unavailable in four separate stories.** 6.4, 6.5, 6.6, 6.7 all note "the Chrome extension was not connected this session" and fell back to chrome-devtools MCP / Playwright MCP. In 6.5 the browser-visual poll-observation pass was outright blocked (covered instead by fake-timer DOM tests + an HTTP-level proxy smoke). Fallbacks worked, but it is recurring tooling friction that cost coverage.

3. **The model-per-job convention drifted, silently.** CLAUDE.md says Sonnet 5 xHigh for dev-story. 6.1 ran on Sonnet 5 xHigh (correct), but 6.5 explicitly records "Story record recommends Sonnet 5 xHigh; run was invoked on Opus 4.8," and 6.8 likewise ran on Opus 4.8. Not a defect (both models are fine), but the documented convention wasn't followed and it surfaced only in the record, never as a decision.

4. **The row gesture model churned across three stories before settling.** 6.3: single-click selects / second-click opens. 6.6: added single-click-Title-cell → edit. 6.7 was PLANNED (in 6.6's Dev Notes and epics.md) to switch to double-click-to-open, but was superseded mid-epic by a hover Open button, leaving arm-select as the inline-edit gate. A good end state, but three revisions of one interaction inside one epic, and the plan predicted a different one.

5. **Opportunistic bundling / plan-vs-shipped drift continued (mild).** 6.3's round-3 added `user-select:none` for a double-click text-flash fix; 6.7 folded in the 2026-07-05 hover-Open UX change. Sound process each time, but — as in Epic 3 — `epics.md`'s story text no longer fully describes the shipped gesture model.

## Key takeaways

1. **Green unit suites are necessary, never sufficient.** The epic's three worst bugs (6.1 app-crash on unmount, 6.4 StrictMode dropping every upload, 6.5 Crossref mis-correction) were all invisible to a fully green suite and surfaced only in live smoke. jsdom's `render()` not being StrictMode-wrapped is a standing blind spot — keep smoking real conditions.
2. **Close a deferred item ahead of the feature that needs it.** AE-4 doc-scoping was closed by Story 5.8 BEFORE Epic 6, and 6.7's isolation is the clean payoff — the exact inverse of Epic 3's defer-and-get-bitten. This is the pattern to repeat for any known debt a coming feature will lean on.
3. **Structural-refactor-as-a-story is now a proven epic-close move.** 6.8 (echoing 5.0) ended the epic on a clean modular footing behind a byte-identical contract. Keep sequencing a refactor story at epic boundaries once the god-modules have grown.
4. **A carry-over item that keeps surviving retros may be hard, not neglected.** The DPR>1 cross-page smoke backfill has now outlived two retros; Epic 6 showed the real reason (cross-page drag is un-automatable in this harness). Name the true difficulty and adopt the workaround as standard, rather than re-listing it as a simple to-do.
5. **Artifact hygiene held; the convention that slipped was model-per-job.** Dev Agent Records were complete across all 8 stories (AE3-2's fix stuck). The next cadence to tighten is the model/doc alignment (AE6-3), not the records.
